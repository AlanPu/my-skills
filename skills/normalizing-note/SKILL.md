# normalizing-note
## 指令
### 1. 获取内容
- 根据上下文，获取到笔记的原始内容
### 2. 理解内容
- 识别原始的主题和核心观点
- 发现内容之间的逻辑关系
- 确定用户的思维模式和表达习惯
- 理解上下文和背景信息
### 3. 结构化整理
- 根据内容逻辑，建立合适的层次结构
- 将相关观点归类整合
- 保留用户的思考轨迹和关键表达
- 使用恰当的 Markdown 语法组织内容
### 4. 内容优化
- 删除重复和无意义的冗余信息
- 修正明显的错别字和语法错误
- 补充必要的连接词和过渡语，使行文更流畅
- 突出重要观点和关键信息
### 5. 格式化输出
- 选择合适的标题层级（#、##、### 等）
- 使用列表、引用、加粗等格式提升可读性
- 保持简洁清晰的版面布局
- 确保输出格式符合 Markdown 规范
- 标题列表等不要使用数字编号
### 6. 保存结果
- 新建一个.md文件保存修改后的结果。文件名格式按下面两条规则执行
- 如果原文件不是.md格式，则按源文件名格式，并改成.md文件保存，例如：原文件叫free-note.txt，输出文件名应该是free-note.md
- 如果原文件已经是.md格式，则添加"-formatted"的后缀，例如：原文件叫free-note.md，输出文件名应该是free-note-formatted.md
## 约束
- 保留用户的核心观点和独特表达方式
- 不擅自添加原文没有的信息
- 重要的数据、引用、结论必须保持原意
- 对于不确定的内容，保持谨慎处理
- 尊重用户的思维习惯和表达风格
- 如果原内容已经有部分是标准Markdown格式，那么判断层级是否有明显问题
    - 如果有明显问题，那么重新规范化
    - 如果没有明显问题，那么不要修改原层级
## 示例
### 原始内容
```
笔记要不要像人脑记忆一样 不用归档
就像ChatGPT有历史对话
但是这个历史可以按项目分类
这样是不是就够了
对了 还要考虑搜索功能
标签系统可能也有用
```
### 输出结果
```
# 笔记的归档与管理
## 核心思考
笔记系统是否可以模仿人脑记忆机制，无需复杂的归档过程？
## 参考案例
ChatGPT 的历史对话机制提供了类似的参考价值
## 解决方案
### 项目分类
- 将笔记按照项目进行分组管理
- 简化归档复杂度
- 保持内容的可追溯性
### 辅助功能
- **搜索功能**：快速定位所需内容
- **标签系统**：提供多维度分类方式
## 结论
通过项目分类结合搜索和标签功能，可以实现既简单又高效的笔记管理方案。
```
